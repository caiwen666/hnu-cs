# 浮点数累加精度问题

## 1. 现象

### 1.1 Example A

```rust
let mut sum: f32 = 0.0;
for i in 0..25000000 {
    sum += 1.0;
}
println!("sum: {}", sum);
```

**Output** 16777216.0

**Expected** 25000000.0 

---

![](https://api.file.caiwen.work/picture/2026/03/24/func1_compare.svg)

x 轴：迭代次数

红色：`(i + 1) as f32`

蓝色：`sum`

从某次迭代开始，`sum` 不再发生变化。

### 1.2 Example B & C

```rust
let mut sum: f32 = 0.0;
for i in 0..90000000 {
    sum += 1.03;
}
```

Example B，加 $25000000$ 次：

**Output** 33254464.0

**Expected** 25750000.0

Example C，加 $90000000$ 次：

**Output** 33554432.0

**Expected** 92700000.0

---

![](https://api.file.caiwen.work/picture/2026/03/24/func3_compare.svg)

x 轴：迭代次数

红色：`(i + 1) as f32 * 1.03`

蓝色：`sum`

迭代大概 $\ge 1.6\times 10^7$ 时开始，`sum` 值增长速度突然变快

而迭代大概 $\ge 3.2\times 10^7$ 时开始，`sum` 不再发生变化。

## 2. 分析

### 2.1 Example A

$$
\log_2{25000000} \approx 24.575425
$$

而根据 IEEE754 ，单精度浮点数的尾数只有 $23$ 位，加上隐藏位也只有 $24$ 位，无法表示出 $25000000$。

当 $sum \le 2^{24}-1$ 时，没有精度损失：

```rust
let mut sum: f32 = 0.0;
for _ in 0..16777215 {
    sum += 1.0;
}
println!("sum: {}", sum); // <- sum: 16777215 = 2^24 - 1
```

$2^{24} - 1 = $ 0 10010110 11111111111111111111111

再 $+1$，此时得到 

$2^{24} = $ 0 10010111 00000000000000000000000

再 $+1$ 时，$1.0$ 的尾数在对阶时要右移 $24$ 位，此时 $1.0$ 的隐藏位与 $2^{24}$ 的 G 位对齐：

0 10010111 00000000000000000000000 000

​                                                                         100

相加：

0 10010111 00000000000000000000000 100

向偶数舍入：

0 10010111 00000000000000000000000

导致根本没加上去。因此 $sum$ 加到 $2^{24}$ 就停止。$2^{24} = 16777216$。

### 2.2 Example B & C

$1.03 = $ 0 01111111 00000111101011100001010

> Tips：1.03 的分母不是 $2$ 的幂次，因此无法用浮点数精准表示。

相加时，先对阶再把尾数相加。

某个时刻，对阶之后：

1xxxxxxxxxxxxxxxxxxxxxxx

​                      100000111101011100001010

设当前 $sum$ 的阶码为 $e$。那么 $1.03$ 在对阶时需要右移 $e$ 位。考虑尾数相加这个过程，下面的尾数对最终结果的贡献：

* $1.03$ 的最高有效位（即隐藏位）的贡献，即 $2^{-e+e}=1$。
* 考虑舍入的贡献：
  * 如果向下舍入，则无贡献
  * 如果向上舍入，则相当于加了一个 $ULP$，即 $2^{-23+e}$
* 为了简单起见，我们同时不考虑 $1.03$ 最高位以外的位的贡献，以及被移出尾数部分的位的贡献，以及后续可能的左规的舍入的贡献。

$e=0$ 时：

1xxxxxxxxxxxxxxxxxxxxxxx000

100000111101011100001010

无舍入

$e=1$ 时：

1xxxxxxxxxxxxxxxxxxxxxxx000

 100000111101011100001010

结果的 $G=0$，向下舍入。

$e=2$ 时：

1xxxxxxxxxxxxxxxxxxxxxxx000

 100000111101011100001010

结果的 $G=1$，但 $R$ 和 $S$ 均为 $0$，向偶舍入。

$e=3$ 时：

1xxxxxxxxxxxxxxxxxxxxxxx000

  100000111101011100001010

结果的 $G=0$，向下舍入。

$e=4$ 时：

1xxxxxxxxxxxxxxxxxxxxxxx000

  100000111101011100001010

结果的 $G=1$，同时 $S=1$，向上舍入。

注意到，此时 $1.03$ 尾数的最低 $1$ 位进入了 $S$，因此后续的过程中 $S$ 均为 $1$。

那么后续的舍入情况均只由 $G$ 位决定。$G=1$ 时必然是向上舍入。

于是只需要观察 $1.03$ 剩余尾数的二进制表示中，连续 $0$ 块或是连续 $1$ 块：

| $e$        | 舍入情况 |
| ---------- | -------- |
| $[5, 8]$   | 向下     |
| $[9, 11]$  | 向上     |
| $12$       | 向下     |
| $13$       | 向上     |
| $14$       | 向下     |
| $[15, 18]$ | 向上     |
| $[19,23]$  | 向下     |
| $24$       | 向上     |
| $\ge 25$   | 向下     |

注意，当 $e=18$ 时：

1xxxxxxxxxxxxxxxxxxxxxxx000

​         100000111101011100001010

此时其他位都被移出去了，只有最高有效位的贡献。

$e=24$ 时：

1xxxxxxxxxxxxxxxxxxxxxxx000

​            100000111101011100001010

最高有效位也被移出去了，此时只有舍入的贡献。

$e$ 太小的时候，最后总贡献非常接近于 $1.03$，看起来没意思。我们只考虑 $e$ 较大的几段：

$e=18$ 时，只有最高有效位的贡献，同时向上舍入，有舍入贡献，总贡献为 $1+2^{18-23} = 1+\frac{1}{32} = 1.03125$。

这意味着 $2^{18} \le sum < 2^{19}$ 时，每次加 $1.03$ 实际上加上去的是 $1.03125$。

$e\in [19,23]$ 时，只有最高有效位的贡献，同时向下舍入，没有舍入贡献，总贡献为 $1$。$2^{19} \le sum < 2^{24}$ 时，每次实际上加的是 $1$。

$e=24$ 时，最高有效位贡献也没了，只有舍入贡献，总贡献为 $2^{24-23}=2$，每次实际上加的是 $2$。这也就解释了为什么 $sum$ 有一段增高的特别快。

$e\ge 25$ 时，最高有效位贡献和舍入贡献都没了，总贡献为 $0$，于是什么都没加上。这也就解释了为什么 $sum$ 后面一段没有任何增长。

>事实上，我们有：
>
>* 对于两个单精度浮点数**相加**，如果两个操作数阶码之差的绝对值 $\ge 25$，则结果直接取阶码较大的操作数。
>* 而如果是两个单精度浮点数**相减**，那么阶码之差绝对值要 $\ge 26$ 才能使得结果直接取阶码较大的操作数。
>
>证明留作练习。

## 3. 解决

>如果上面三个例子的迭代次数都不超过 $10^{10}$，那么如何调整阶码和尾数的位数来使得不会出现上述情况。

### 3.1 Example A

$$
\log_2 10^{10} \approx 33.21928
$$

$10^{10}$ 需要 $34$ 个二进制位存下。考虑到隐藏位，浮点数的尾数位数至少需要 $33$。

### 3.2  Example B & C

$$
\log_2 (1.03 \times 10^{10}) \approx 33.261925
$$

所以也需要 $34$ 个二进制位存下。考虑到隐藏位，浮点数的尾数位数至少需要 $33$。

并且显然地，不会出现增长速度突然加快的情况。
